Proyectos

  • Funkoshop

    Codo a codo - 2023

    Motivación
    • Desarrollo de aplicación web que simula el comportamiento de una tienda de Funkos y productos relacionados como remeras o llaveros de los mismos, atendiendo las necesidades de un comprador o administrador de la misma página.
    Funcionalidades
    • Manejo y validación del registro y login de las cuentas de administradores.
    • Administración de los productos almacenados (alta, baja y modificación de estos).
    • Manejo de las compras de los productos
    • Uso de filtros para los productos tanto en la vista de comprador como la de adminastrador.
    • Paginación de la muestra de productos tanto en la vista de comprador como la de adminastrador.
    Stack tecnológico
    • Lenguaje de programación principal: JavaScript
    • Framework: Express
    • Template engine: EJS
    • Gestor de dependencias: npm
    • Motor de base de datos: MySQL

  • RealTime

    Diseño de sistemas - 2023

    Motivación
    • Desarrollo de aplicación web que permite monitorear el estado de la prestación de servicios que resultan necesarios a las diferentes comunidades con movilidad reducida, y son proporcionados en diferentes establecimientos de determinadas entidades.
    Funcionalidades
    • Gestión de comunidades de cada usuario.
    • Apertura de incidentes en todas las comunidades, con una administración aislada de los mismos por cada comunidad.
    • Notificación de aperturas y revisión de incidentes hacia los compañeros de la comunidad del autor del incidente notificado.
    • Manejo de login y registro, y validación de credenciales.
    • Gestión de roles.
    • Administración de usuarios para el rol de adminastrador.
    • Consumos de API Rest:
      • Georef: Obtención de las provincias, municipios y departamentos para gestionarlos con cada usuario.
      • Calculador de puntaje: Actualizar semanalmente el puntaje de cada comunidad y miembro de la misma, condicionandolos por el mismo.
      • Fusión de comunidades: Fusionar comunidades, que son posibles de acuerdo a un análisis previo de la propia API Rest consumida.
    • Actualización de intereses (incidentes a los que fue involucrado) de cada usuario.
    • Administración de entidades prestadoras y organismos de control mediante cargas de archivos.
    • Generación de rankings de los establecimientos y entidades respecto a los incidentes producidos.
    Stack tecnológico
    • Lenguaje de programación principal: Java
    • Framework: Javalin
    • Template engine: Handlebars
    • Gestor de dependencias: Maven
    • ORM: Hibernate
    • Motor de base de datos: PostgreSQL
