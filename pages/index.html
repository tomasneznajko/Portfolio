  • T.U.K.I (The Ultimate Kernel Implementation)

    Sistemas operativos - 2023

    Motivación
    • Simulación de un sistema sistema distribuido, donde se planifican procesos, se resuelven peticiones al sistema y se administran de manera adecuada una Memoria y un sistema de archivos bajo determinados esquemas.
    Stack tecnológico
    • Lenguaje de programación principal: C
    • Framework: Valgrind (Memcheck + Helgrind)
    • Automatizador de compilación: Makefiles
    Links
    • Repositorio
    Funcionalidades generales

    • Levantar archivos de configuración con la información necesaria para ejecutar las funcionalidades particulares de cada módulo.

    • Usar Commons Library para el manejo de estructuras de datos, archivos de configuración y logs.

    • Definir protocolo de comunicación que permita el envío de paquetes, que deben serializarse al recibirse, utilizando arquitectura cliente-servidor entre diferentes procesos de la misma red.

    • Envío de diferentes tipos de logs que informen el estado actual del procesamiento en un módulo particular.

    Módulos

    • Consola

      • Levantar e interpretar archivo con pseudocódigo para generar el listado de instrucciones de un proceso.

      • Conectarse con Kernel, enviarle el listado de instrucciones y esperar la finalización de su procesamiento.

    • Kernel

      • Estrategia de ejecución multihilos para mantener una conexión activa con cada una de las instanciaciones de los otros módulos

      • Administrar los procesos lanzados por medio de las consolas utilizando la estructura PCB, donde se almacenará información administrativa y el contexto de ejecución del proceso. Se envia a CPU al ejecutar el proceso.

      • Crear planificador de largo plazo: Planificar los procesos en los estados New y Exit, donde se comunica con Memoria para inicializar o liberar los recursos asignados a estos.

      • Crear planificador de corto plazo: Planificar los procesos en los estados Ready, Block y Exec, empleando el algoritmo HRRN o FIFO. Mantener una comunicación activa con CPU.

      • Gestionar los recursos determinados en archivo de configuración y utilizados en las ejecuciones de los procesos.

      • Administrar el espacio brindado en Memoria. Solicitar creación o eliminación de segmentos a Memoria.

      • En caso de compactación en Memoria, verificar que no haya operaciones en File System o Memoria para iniciar la actualización de PCB al recibir tabla de segmentos actualizada por Memoria.

      • Orquestar las llamadas a File System, implementando una tabla global de archivos abiertos y bloqueos exclusivos y obligatorios para la gestión de los mismos.

    • CPU

      • Interpretar y ejecutar las instrucciones de los Contextos de Ejecución recibidos desde Kernel, simulando un ciclo de instrucción simplificado.

      • Traducir direcciones lógicas del proceso a direcciones físicas de Memoria. Implementación MMU.

      • Actualizar Contextos de Ejecución y enviarlos hacia Kernel, incluyendo registros de propósito general.

    • Memoria

      • Responder los pedidos realizados por la CPU para leer y/o escribir en los segmentos de datos del proceso en ejecución.

      • Crear y gestionar las estructuras administrativas necesarias al establecer conexión con Kernel, CPU y File System: Espacio contiguo de memoria asignado al usuario para que los procesos lean y escriban, y estructuras auxiliares para la implementación de segmentación (tablas de segmentos y lista de huecos libres).

      • Manejar esquema de segmentación, utilizando los algoritmos first fit, best fit y worst fit para la ubicación de los segmentos en los espacios libres.

      • Simular velocidad de accesos a Memoria dando un tiempo de espera ante solicitudes de acceso al espacio de usuario de CPU o Kernel.

      • Implementar proceso de compactación al contar con espacio necesario para la creación de un segmento solicitado, requiriendo de juntar huecos libres para asignar un espacio contiguo e informar la actualización de la tabla de segmentos a Kernel.

    • File System

      • Persistir información por medio de archivos configurados dentro del archivo de configuración.

      • Levantar bitmap de bloques y recorrer directorio de FCBs para crear estructuras administrativas que permita administrar los archivos al haber establecido comunicación con Memoria como cliente.

      • Atender las solicitudes del Kernel, como servidor, mediante las operaciones abrir, crear, truncar, leer y escribir un archivo.

      • Mantener actualizadas en disco todas las operaciones realizadas sobre los FCBs, Bitmap y Bloques.
